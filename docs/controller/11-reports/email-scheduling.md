# Email Scheduling

## Overview

Email Schedules automatically generate a report from a template and send it to a configured list of recipients on a recurring basis. Once set up, reports arrive in recipients' inboxes without any manual action — ideal for weekly executive summaries, monthly SLA reviews, or daily operational digests.

Navigate to **Report > Email Schedules** to manage schedules.

## Prerequisites

Before creating an email schedule:

- You must have at least one report template saved. See [Report Templates](templates.md).
- SMTP must be configured on the controller. Contact your administrator if outbound email is not working.
- All intended recipients must have valid email addresses.

## Creating a Schedule

1. Navigate to **Report > Email Schedules**.
2. Click **Add Schedule**.
3. Complete the schedule form:

| Field | Description |
|-------|-------------|
| **Schedule Name** | A descriptive label (e.g., "Weekly SLA Report — Mumbai") |
| **Template** | Select the report template to use |
| **Frequency** | Daily, Weekly, or Monthly |
| **Run Time** | The time of day to generate and send the report (24-hour format) |
| **Day of Week** | (Weekly only) Select Monday–Sunday |
| **Day of Month** | (Monthly only) Select 1–28 (use 28 to avoid end-of-month edge cases) |
| **Recipients** | One or more email addresses (comma-separated) |
| **Subject** | Email subject line — supports template variables |
| **Body** | Email body text — supports template variables and basic markdown |
| **Timezone** | Timezone for the run time (defaults to organization timezone) |

4. Click **Save**. The schedule is active immediately and will run at the next scheduled time.

## Frequency Options

### Daily

The report is generated every day at the configured run time. The report date range is the previous calendar day (midnight to midnight).

**Example:** Run at 06:00 Monday–Sunday to get yesterday's traffic summary each morning.

### Weekly

The report runs once per week on the selected day at the configured time. The report date range is the previous 7 days.

**Example:** Run at 08:00 every Monday to get last week's SLA compliance report for your team's weekly meeting.

### Monthly

The report runs once per month on the selected day of month. The report date range is the previous calendar month (1st to last day).

**Example:** Run on the 1st at 09:00 to get last month's data usage report for billing reconciliation.

## Email Subject and Body Variables

Use template variables to make the email subject and body dynamic:

| Variable | Example output |
|----------|---------------|
| `{{topology_name}}` | Mumbai-Region |
| `{{start_date}}` | 2024-06-01 |
| `{{end_date}}` | 2024-06-30 |
| `{{device_count}}` | 12 |
| `{{org_name}}` | Acme Corp |
| `{{generated_at}}` | 2024-07-01 09:00 IST |

Example subject: `[{{org_name}}] Monthly Traffic Report — {{start_date}} to {{end_date}}`

## Schedule Status Table

The **Email Schedules** list shows a status column for each schedule:

| Status | Description |
|--------|-------------|
| **Active** | Schedule is running normally |
| **Last Run: Success** | Most recent send completed successfully (timestamp shown) |
| **Last Run: Failed** | Most recent send failed; retry pending |
| **Paused** | Schedule is disabled; will not run until resumed |

Click any schedule name to see the full run history, including timestamps and delivery status for each past send.

## Retry Behavior

If a report send fails (SMTP error, temporary network issue, or report generation error), the scheduler automatically retries:

- **Retry 1** — 1 hour after the original failure
- **Retry 2** — 2 hours after the original failure
- **Retry 3** — 4 hours after the original failure

If all three retries fail, the send is marked as permanently failed and will not retry until the next scheduled run. An alert email is sent to the administrator's email address when a schedule fails all retries.

## Pausing and Resuming a Schedule

To temporarily pause a schedule (for example, during system maintenance):

1. Click the schedule name.
2. Click **Pause Schedule**.
3. The schedule stops running. It will not send missed reports for the paused period.
4. To resume, click **Resume Schedule**.

## Sending a Test Email

Before your schedule's first run, send a test to verify formatting and delivery:

1. Open the schedule detail page.
2. Click **Send Test Now**.
3. The report is generated immediately using the most recent available data and sent to all configured recipients.
4. Check that the email arrives and the attachment opens correctly.

## Viewing Generated Reports

Every report generated by a schedule is stored on the controller for 7 days:

1. Navigate to **Report > Email Schedules**.
2. Click a schedule name.
3. In the **Run History** tab, click **Download** next to any completed run to download the report file.

## Troubleshooting

| Symptom | Cause | Resolution |
|---------|-------|------------|
| Schedule shows "Last Run: Failed" | SMTP error or report generation error | Check the error details in the run history; verify SMTP settings with your administrator |
| Recipients not receiving emails | Email in spam folder or wrong address | Ask recipients to check spam; verify address spelling in schedule settings |
| Report arrives with no data | Date range has no data for the selected scope | Verify the template scope covers devices that were active during the report period |
| Schedule ran at the wrong time | Timezone mismatch | Check the organization timezone setting and the schedule's timezone field |
| Schedule silently stops running | Organization subscription or storage limit reached | Contact your administrator to review storage usage |
